# Frontend Design Skill Summary

This skill guides creation of **distinctive, production-grade frontend interfaces** that avoid generic AI aesthetics.

## Key Principles

**Design Thinking First**: Before coding, establish a bold aesthetic direction by understanding purpose, choosing an extreme tone, identifying constraints, and determining what makes the interface unforgettable.

**Core Implementation Focus**:
- "Production-grade and functional" interfaces that are "visually striking and memorable"
- Cohesive aesthetic with clear point-of-view
- Meticulous attention to refined details

## Critical Guidelines

**What to Embrace**:
- Distinctive, characterful typography (avoid generic fonts)
- Cohesive color palettes with dominant colors and sharp accents
- High-impact animations and scroll-triggered effects
- Unexpected layouts with asymmetry, overlap, and negative space
- Atmospheric backgrounds and contextual visual details

**What to Avoid**:
- Generic aesthetics (overused fonts, predictable layouts)
- "Cookie-cutter design" lacking context-specific character
- Clichéd color schemes and standard component patterns

## Core Philosophy

Design intentionality matters more than intensity—both "bold maximalism and refined minimalism" work when executed with precision for the specific context. Match implementation complexity to your aesthetic vision: elaborate code for maximalist designs, careful restraint for minimalist ones.
